# 9 DoR e DoD

## 9.1 Definition of Ready (DoR)

O DoR define tudo aquilo que precisa ser verificado para que cada user story esteja pronta para ser desenvolvida, tendo isso em mente, utilizamos os seguintes critérios:

- **Valor Claro:** A história define claramente o perfil, a função e o seu valor, sem ambiguidades.
- **Entendimento rápido:** É possível para a equipe entender a história de forma simples e rápida.
- **Critérios de aceitação:** Toda história de usuário é acompanhada por critérios de aceitação.
- **Tamanho adequado:** A história tem tamanho compatível com uma sprint e pode ser concluída dentro dela.

## 9.2 Definition of Done (DoD)

Para definir que a funcionalidade realmente seja considerada completa, no nosso projeto são utilizados os seguintes critérios:

- Código desenvolvido e revisado (Code Review)
- Testes unitários e de integração passando
- Testes de aceitação validados 
- Código integrado ao nosso repositório principal
- Funcionalidade validada pelo Product Owner.
